# راهنمای تم‌بندی (Theming)

شیرازه از یک سیستم تم‌بندی قدرتمند و انعطاف‌پذیر مبتنی بر متغیرهای CSS (CSS Variables) استفاده می‌کند. این سیستم به شما اجازه می‌دهد تا به راحتی ظاهر ویکی خود را شخصی‌سازی کرده یا حتی تم‌های کاملاً جدیدی ایجاد کنید.

## مفاهیم پایه

-   **تم روشن و تاریک:** شیرازه به صورت پیش‌فرض دو تم داخلی دارد: `light` و `dark`.
-   **تشخیص خودکار:** برنامه می‌تواند تم سیستم‌عامل کاربر را تشخیص داده و به صورت خودکار یکی از این دو تم را به عنوان پیش‌فرض انتخاب کند.
-   **کنترل کاربر:** یک دکمه در هدر سایدبار به کاربر اجازه می‌دهد بین تم‌ها جابجا شود. انتخاب کاربر در حافظه مرورگر (`localStorage`) ذخیره می‌شود و در بازدیدهای بعدی اعمال می‌گردد.
-   **مبتنی بر `data-attribute`:** تم فعال با استفاده از یک `data-attribute` روی تگ `<html>` تنظیم می‌شود (مثلاً `<html data-theme="dark">`). این به این معناست که تمام استایل‌های تم با انتخاب‌گرهایی مانند `[data-theme="dark"]` اعمال می‌شوند.

## شخصی‌سازی سریع رنگ‌ها

ساده‌ترین راه برای تغییر ظاهر، بازنویسی متغیرهای رنگی از طریق فایل `config/config.js` است. کافیست نام متغیر را به صورت `camelCase` در بخش `theme` وارد کنید.

**مثال: تغییر رنگ اصلی به نارنجی و رنگ متن به خاکستری تیره‌تر**

```javascript
// in config/config.js
theme: {
  default: 'auto',
  primaryColor: '#f97316', // Orange 500
  textColor: '#1f2937',    // Gray 800
}
```

این تغییرات هم در تم روشن و هم در تم تاریک اعمال می‌شوند و به شما اجازه می‌دهند هویت بصری برند خود را به سادگی پیاده کنید.

## لیست متغیرهای رنگی اصلی

در جدول زیر، لیست کامل متغیرهای رنگی که می‌توانید در `config.js` بازنویسی کنید، آمده است.

| نام متغیر (در `config.js`) | نام متغیر CSS    | توضیحات                                       |
| --------------------------- | ----------------- | ---------------------------------------------- |
| `bgColor`                   | `--bg-color`      | رنگ پس‌زمینه اصلی صفحه                         |
| `sidebarBg`                 | `--sidebar-bg`    | رنگ پس‌زمینه منوی کناری                       |
| `contentBg`                 | `--content-bg`    | رنگ پس‌زمینه بخش محتوا                       |
| `textColor`                 | `--text-color`    | رنگ اصلی متن پاراگراف‌ها                      |
| `textColorStrong`           | `--text-color-strong` | رنگ متن برای عناصر مهم‌تر مانند `<strong>` |
| `primaryColor`              | `--primary-color` | رنگ اصلی برای لینک‌های فعال، عناوین و تأکیدها    |
| `secondaryColor`            | `--secondary-color` | رنگ دوم برای لینک‌های داخل محتوا             |
| `borderColor`               | `--border-color`  | رنگ جداکننده‌ها و حاشیه‌ها                     |
| `codeBg`                    | `--code-bg`       | رنگ پس‌زمینه بلوک‌های کد (`<code>`)          |
| `codeText`                  | `--code-text`     | رنگ متن بلوک‌های کد                           |
| `quoteBorder`               | `--quote-border`  | رنگ خط کنار نقل‌قول‌ها (`<blockquote>`)   |
| `quoteText`                 | `--quote-text`    | رنگ متن نقل‌قول‌ها                           |
| `errorBg`                   | `--error-bg`      | رنگ پس‌زمینه پیام‌های خطا                     |
| `errorBorder`               | `--error-border`  | رنگ حاشیه پیام‌های خطا                       |
| `errorText`                 | `--error-text`    | رنگ متن پیام‌های خطا                         |

## ساخت یک تم سفارشی

اگر شخصی‌سازی رنگ‌ها کافی نیست و می‌خواهید یک تم کاملاً جدید (مثلاً `sepia`) بسازید، مراحل زیر را دنبال کنید:

1.  **ایجاد فایل CSS:** یک فایل CSS جدید در پروژه خود بسازید (مثلاً `assets/themes/sepia.css`).

2.  **تعریف متغیرها:** در این فایل، تمام متغیرهای رنگی لیست شده در جدول بالا را با استفاده از انتخاب‌گر `[data-theme="sepia"]` تعریف کنید.

    ```css
    /* in assets/themes/sepia.css */
    [data-theme="sepia"] {
      --bg-color: #f4ecd8;
      --text-color: #5b4636;
      --primary-color: #8c6f5a;
      /* ... and all other variables */
    }
    ```

3.  **فراخوانی فایل CSS:** فایل CSS جدید خود را در `index.html` فراخوانی کنید.

    ```html
    <!-- in index.html, inside <head> -->
    <link rel="stylesheet" href="src/styles/main.css">
    <link rel="stylesheet" href="assets/themes/sepia.css"> <!-- Add this line -->
    ```

4.  **فعال‌سازی تم:** برای فعال‌سازی تم جدید، باید به صورت دستی `data-attribute` را تغییر دهید. این کار در حال حاضر از طریق `config.js` پشتیبانی نمی‌شود و نیاز به تغییرات در کد جاوااسکریپت دارد، اما این ساختار پایه به شما اجازه می‌دهد تا به سادگی این قابلیت را در آینده اضافه کنید.
